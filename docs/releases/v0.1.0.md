# Patch Courier v0.1.0

Initial open-source release of Patch Courier.

## What is included

- Native macOS app target: `PatchCourierMac`.
- Long-lived daemon target: `MailroomDaemon` / `mailroomd`.
- Local `codex app-server` integration over stdio JSON-RPC.
- SQLite persistence for threads, turns, approvals, events, mailbox sync cursors, mailbox accounts, and sender policies.
- Keychain-backed mailbox password storage.
- Local daemon control plane for state reads, approval resolution, and daemon-owned config mutation.
- HTML and plain-text outbound email rendering for receipts, first-contact decisions, project selection, approvals, completion, failure, and saved-for-later states.
- Mail preview fixture renderer: `./scripts/render_mail_previews.sh`.
- Unit tests for approval mail codec behavior and mail parser behavior.
- Open-source project hygiene: MIT license, contribution guide, code of conduct, security policy, and GitHub Actions build workflow.

## Known limitations

- This is still a pre-1.0 prototype; storage schema and internal names may change.
- First-run setup is not yet polished for non-contributors.
- The app is not yet signed/notarized for binary distribution.
- Some internal code still uses `Mailroom*` names because the original implementation was mailroom-oriented; public branding is now Patch Courier.
- The token parser accepts both `[patch-courier:...]` and legacy `[codex-mailroom:...]` tokens for compatibility.

## Verification

Validated locally on macOS with Xcode:

```bash
xcodegen generate
xcodebuild -project PatchCourier.xcodeproj -scheme MailroomDaemon -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO test
xcodebuild -project PatchCourier.xcodeproj -scheme MailroomDaemon -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build
xcodebuild -project PatchCourier.xcodeproj -scheme PatchCourierMac -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build
```
