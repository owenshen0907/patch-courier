# Patch Courier v0.2.0

Reliability and recovery release for the email-to-local-Codex loop.

## What changed

- Approval reminder recovery now tracks the approval request identity, so restart/retry paths do not resend the same pending approval email.
- Restarted active mail turns use bounded recovery polling and eventually record a visible `turn/recovery/timeout` system error instead of staying active forever.
- Mailbox replay now persists inbound message handling by `(mailbox_id, uid)` and skips already-processed messages while still retrying unfinished received records.
- Mailbox poll, sync, and history backfill failures are persisted as poll incidents and surfaced in daemon snapshots and the macOS mailbox health UI.
- Mail preview fixtures now have tests covering receipt, managed-project selection, approval request, completion, failure, and saved-for-later flows.
- SQLite schema compatibility now uses `PRAGMA user_version`, migrates legacy version-0 databases in place, and fails closed on databases created by a newer binary.

## Operator impact

- Duplicate approval emails should be materially less likely after daemon restarts or mailbox replay.
- Transport failures are no longer only transient runtime state; the app can show recent incident details after refresh/restart.
- Startup recovery has clearer terminal behavior for stale turns.
- Future stored-state changes have a documented migration path in `docs/STORAGE_MIGRATIONS.md`.

## Verification

Validated locally on macOS with Xcode:

```bash
xcodegen generate
xcodebuild -project PatchCourier.xcodeproj -scheme MailroomDaemon -destination 'platform=macOS' -derivedDataPath /tmp/PatchCourierV02MigrationCheck CODE_SIGNING_ALLOWED=NO test
xcodebuild -quiet -project PatchCourier.xcodeproj -scheme MailroomDaemon -destination 'platform=macOS' -derivedDataPath /tmp/PatchCourierV02MigrationCheck CODE_SIGNING_ALLOWED=NO build
xcodebuild -quiet -project PatchCourier.xcodeproj -scheme PatchCourierMac -destination 'platform=macOS' -derivedDataPath /tmp/PatchCourierV02MigrationCheck CODE_SIGNING_ALLOWED=NO build
```

## Known limitations

- First-run onboarding is still contributor-oriented and moves to v0.3.
- The app is not yet signed/notarized for binary distribution.
- Storage schema is still pre-1.0, but older databases should now upgrade predictably and newer databases are protected from older binaries.
