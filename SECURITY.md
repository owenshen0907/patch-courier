# Security Policy

If you discover a security issue, do not open a public issue with exploit details.

Instead, use GitHub's private vulnerability reporting flow for the repository if it is enabled. If private reporting is not available yet, contact the maintainer through the repository profile and include:

- a clear description of the issue
- impact and affected paths
- reproduction steps or a proof of concept
- any suggested mitigation

We will acknowledge reports as quickly as possible and coordinate a fix before public disclosure when feasible.
